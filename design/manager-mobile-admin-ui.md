# Manager 管理端手机 UI 设计

## 目标

让运营人员在手机上也能完成日常数据管理：浏览、筛选、新建、编辑、启用、删除，而不是只能横向拖宽表。

## 交互原则

1. **一屏一事**：窄屏不并排双栏；列表与编辑改为钻取（选中 → 进入详情 → 返回）。
2. **关键信息优先**：卡片首行展示标题/主标识 + 状态；次要字段折叠为两列 meta。
3. **操作收纳**：多操作收入「⋯」菜单；主操作（新建）保留在页头。
4. **编辑全屏感**：对话框在手机上贴边、可接近全高，底部操作栏固定，适配 safe-area。
5. **导航完整**：移动抽屉补齐退出登录、管理员信息、版本号。

## 断点

| 断点 | 行为 |
|------|------|
| `< md` (768px) | 顶栏 + 抽屉导航；列表用卡片；对话框全宽；系统变量钻取 |
| `md+` | 侧栏 + 表格（与现网一致） |
| `< lg` | 跟读管理：分类 ↔ 词汇钻取 |
| `< xl` | 数据备份 / 服务器商店保持纵向堆叠（已有） |

## 组件策略

| 组件 | 手机表现 |
|------|----------|
| `AdminTable` | 有 `#mobile` 槽时隐藏宽表，渲染卡片列表；否则保留横向滚动兜底 |
| `AdminMobileCard` / `AdminMobileMeta` | 统一卡片骨架 |
| `AdminOverflowMenu` | 行内操作菜单 |
| `AdminDialog` | 接近全屏、footer 可换行、safe-area |
| `SystemVariablesPanel` | 列表 → 编辑钻取 |
| `read-aloud` | 分类 → 词汇钻取 |

## 页面覆盖

全部实际入口：仪表盘、用户、会话、消息、会员、语言、提示词、练习场景、跟读、AI 设置（含语音角色等 tab）、系统设置（含系统变量）、删除记录备份面板。
